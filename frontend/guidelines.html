// HACKATHON_GUIDELINES

01

PARTICIPATION GUIDELINES

// Eligibility and team formation rules for SkilStation.

  • ● PARTICIPANTS MUST BE CURRENTLY ENROLLED IN A RECOGNIZED UNIVERSITY OR COLLEGE.
  • ● TEAMS CAN CONSIST OF 1–4 MEMBERS. INDIVIDUAL REGISTRATIONS ARE ALSO ACCEPTED.
  • ● INTER-COLLEGE TEAMS ARE ALLOWED AND ENCOURAGED.
  • ● EACH PARTICIPANT MAY REGISTER IN ONLY ONE TEAM.
🎓
02

DEVELOPMENT STANDARDS

// Technical expectations for code quality, security, and documentation.

  • ● MAINTAIN CLEAN, MODULAR, AND READABLE CODE THROUGHOUT THE PROJECT.
  • ● SECURE ALL API KEYS AND TOKENS USING ENVIRONMENT VARIABLES (E.G., .ENV FILES).
  • ● PROVIDE A CLEAR README EXPLAINING SETUP, ARCHITECTURE, AND EXECUTION STEPS.
  • ● ENSURE THAT ALL SOURCE CODE SHARED WITH JUDGES CONTAINS NO PERSONAL OR SENSITIVE DATA.
🏗️
03

PROJECT SUBMISSION

// Requirements for submitting your project for final evaluation.

  • ● SUBMIT YOUR IDEA AND PROPOSED SOLUTION BEFORE 27TH APRIL 2026 TO GET SHORTLISTED FOR THE SUMMER HACKATHON.
  • ● A SHORT 2-MINUTE DEMO VIDEO EXPLAINING THE SYSTEM AND ITS FUNCTIONALITY FOR BETTER UNDERSTANDING AND EVALUATION.
  • ● PROVIDE PROPER DOCUMENTATION AND SETUP README OR LIVE LINK FOR THE PROJECT.
  • ● TOP TEAMS WILL BE SELECTED FOR THE SUMMER HACKATHON FINALE ON 14–15 MAY 2026.
🏅
04

RULES & CONDUCT

// Mandatory guidelines to maintain fairness and professionalism during the hackathon.

  • ● COPIED PROJECTS WILL RESULT IN IMMEDIATE DISQUALIFICATION.
  • ● PARTICIPANTS MUST MAINTAIN RESPECTFUL AND PROFESSIONAL BEHAVIOR AT ALL TIMES.
  • ● FULLY PRE-BUILT OR PREVIOUSLY COMPLETED PROJECTS ARE NOT ALLOWED. ONLY OPEN-SOURCE LIBRARIES AND FRAMEWORKS MAY BE USED.
  • ● ONLY UNIQUE IDEA AND SOLUTION WILL GET MORE CHANCE FOR SELECTION FOR THE SUMMER HACKATHON FINALE.
